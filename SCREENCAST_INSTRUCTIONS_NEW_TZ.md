# Инструкции для скринкастов - Новые пункты ТЗ

Документация по записи скринкастов для новых требований ТЗ: иерархические категории, город в cookie, SMS авторизация, метрики в товарах.

---

## 📌 ПУНКТ 1: Иерархическое дерево категорий с аккордеоном

### Описание
На главной странице сайта отображаются популярные категории в карточках, а также полное дерево категорий с подкатегориями в "аккордеон" стиле - категории скрывают/показывают свои подкатегории при клике.

### Где проверять
- **Главная страница** - http://localhost:3001
- **Раздел "Популярные категории"** - видны 7 категорий в сетке
- **Раздел "Все категории и подкатегории"** - внизу страницы, дерево с иерархией

### Пошаговая инструкция скринкаста

#### ШАГ 1.1: Открыть главную страницу и скроллить к категориям
```
1. Перейти на http://localhost:3001
2. Дождаться загрузки страницы
3. Скроллить вниз примерно на 800-1000px
4. Найти заголовок "Популярные категории" (в сетке 7 категорий)
5. Скроллить дальше вниз
6. Найти заголовок "Все категории и подкатегории" (раздел с деревом)
```

**Ожидаемый результат:**
- Категории отображаются в виде красивых карточек с изображениями
- Дерево категорий видно ниже, показаны основные категории

#### ШАГ 1.2: Проверить аккордеон - раскрыть подкатегории
```
1. В разделе "Все категории и подкатегории" найти категорию, у которой видны подкатегории
   Например: "Электроника", "Мебель", "Одежда" и т.д.
2. Кликнуть на название категории слева (где текст категории или маленький треугольник)
3. Подкатегории должны раскрыться и показать список дочерних категорий
4. Кликнуть снова - подкатегории скроются
```

**Ожидаемый результат:**
- Кликнув на категорию, видна подсписок категорий уровнем ниже
- Повторный клик скрывает этот список
- Визуальный индикатор: треугольник указывает вниз (раскрыто) или вправо (закрыто)

#### ШАГ 1.3: Кликнуть на категорию -> перейти на страницу товаров
```
1. В дереве категорий или в сетке популярных категорий выбрать категорию
2. Кликнуть на название категории (или на карточку в сетке)
3. Браузер должен перейти на /products?global_category_id=N
4. На странице должны отобразиться товары этой категории
```

**Ожидаемый результат:**
- URL: `http://localhost:3001/products?global_category_id=НОМЕР`
- Заголовок страницы изменяется на название категории
- Товары фильтруются по выбранной категории

### Код для проверки (DevTools Console)
```javascript
// Проверить, что категории загружены
const categories = document.querySelectorAll('[class*="category"]');
console.log(`Найдено элементов с категориями: ${categories.length}`);

// Проверить дерево категорий
const categoryTree = document.querySelector('[class*="category-tree"]');
console.log(`Дерево категорий загружено:`, !!categoryTree);
```

## 📌 ПУНКТ 2: Убрать город в выдаче, оставить адрес и город в cookie

### Описание
- **В карточках товаров** - город НЕ показывается (убран из UI)
- **В cookie** - хранится адрес доставки пользователя
- **В localStorage** - гео-данные для быстрого восстановления

### Где проверять
- **Страница товаров** - http://localhost:3001/products
- **DevTools (Application)** - cookies и localStorage

### Пошаговая инструкция скринкаста

#### ШАГ 2.1: Открыть страницу товаров и проверить, что города НЕТ в карточках
```
1. Перейти на http://localhost:3001/products
2. Дождаться загрузки товаров (если загружаются)
3. Найти первую карточку товара
4. Проверить содержимое карточки:
   ✓ Название товара
   ✓ Изображение
   ✓ Цена
   ✓ Рейтинг
   ✓ Кнопка "В корзину"
   ✗ ГОРОД НЕ должен быть показан
   ✗ Адрес НЕ должен быть показан
```

**Ожидаемый результат:**
- Карточка товара содержит только релевантные данные (название, цена, рейтинг, кнопку)
- Никаких городов, адресов в карточке

#### ШАГ 2.2: Проверить, что адрес хранится в cookie
```
1. Кликнуть на кнопку "Укажите адрес доставки" в хедере
2. Выбрать или ввести адрес (например, "Москва")
3. Закрыть модальное окно
4. Открыть DevTools (F12 или Ctrl+Shift+I)
5. Перейти на вкладку "Application" -> "Cookies"
6. Найти cookies для http://localhost:3001
7. Найти cookie с именем "bystroi_address"
```

**Ожидаемый результат:**
- Cookie "bystroi_address" содержит значение адреса (например, "Москва")
- Cookie имеет срок действия 30 дней (max-age=2592000)
- При перезагрузке страницы адрес восстанавливается из cookie

#### ШАГ 2.3: Проверить localStorage для гео-данных
```
1. Открыть DevTools (F12)
2. Перейти на вкладку "Application" -> "Local Storage" -> http://localhost:3001
3. Найти ключ "detected_city"
4. Проверить значение (должен быть JSON с lat/lon)
```

**Ожидаемый результат:**
```json
{
  "lat": 55.7558,
  "lon": 37.6173,
  "city_name": "Москва"
}
```

### Код для проверки (DevTools Console)
```javascript
// Получить адрес из cookie
document.cookie.split('; ').find(c => c.startsWith('bystroi_address'));

// Получить гео-данные из localStorage
const detected = JSON.parse(localStorage.getItem('detected_city'));
console.log('Гео-данные:', detected);

// Проверить, что в карточке товара нет города
const productCards = document.querySelectorAll('[data-testid^="product-"]');
productCards.forEach(card => {
  const text = card.innerText;
  console.log('Содержимое карточки:', text);
  if (text.includes('Москва') || text.includes('Санкт-Петербург')) {
    console.warn('⚠️ ГОРОД НАЙДЕН В КАРТОЧКЕ!');
  }
});
```

## 📌 ПУНКТ 4: Гео пользователя хранить в cookie/localStorage

### Описание
Географические данные пользователя (адрес, координаты) хранятся локально для быстрого восстановления и использования в фильтрах (радиус, доставка).

### Где проверять
- **DevTools (Application)** - Cookies и localStorage
- **Код** - [src/shared/lib/city-utils.ts](src/shared/lib/city-utils.ts)

### Пошаговая инструкция скринкаста

#### ШАГ 4.1: Проверить хранение адреса в cookie при клике "Укажите адрес"
```
1. Перейти на http://localhost:3001
2. Кликнуть на кнопку "Укажите адрес доставки" в хедере (вверху слева)
3. Должно открыться модальное окно с поиском адреса
4. Ввести город (например, "Москва")
5. Выбрать из результатов (если есть)
6. Закрыть модальное окно или кликнуть "Применить"
7. Обновить страницу (F5)
8. Проверить, что адрес остался в хедере
9. Открыть DevTools -> Application -> Cookies
10. Найти cookie "bystroi_address" и проверить значение
```

**Ожидаемый результат:**
- При обновлении страницы адрес восстанавливается из cookie
- Cookie "bystroi_address" содержит выбранный адрес

#### ШАГ 4.2: Проверить sessionStorage для автоматически определенных координат
```
1. Открыть DevTools -> Application -> Session Storage -> http://localhost:3001
2. Найти ключ "detected_city"
3. Проверить значение - должно быть JSON:
   {
     "lat": ЧИСЛО,
     "lon": ЧИСЛО,
     "city_name": "Название"
   }
```

**Ожидаемый результат:**
- `detected_city` содержит координаты города
- Эти координаты автоматически подставляются в фильтры (например, радиус поиска)

#### ШАГ 4.3: Проверить использование гео в радиус-фильтре
```
1. Перейти на http://localhost:3001/products
2. Найти селект "Радиус: 20 км"
3. Изменить радиус (например, на 10 км)
4. Проверить URL - должны быть параметры:
   ?radius_km=10&lat=ЧИСЛО&lon=ЧИСЛО&apply_radius_filter=true
5. Товары должны переисчислиться с учетом нового радиуса
```

**Ожидаемый результат:**
- Координаты (lat, lon) из localStorage автоматически подставляются в URL при выборе радиуса
- API получает правильные координаты для фильтрации

### Код для проверки (DevTools Console)

## 📌 ПУНКТ 5: Поддержка + телефон сверху (фиксированный блок в хедере)

- **Телефон поддержки** - кликабельный номер tel:

- **На мобильных** - tel: это истинная кликабельная ссылка (откроется приложение телефона/мессенджера)

### Где проверять
- **Верхний хедер сайта** - видно на всех страницах
- **Desktop (1440px)** - видны оба варианта (Email и Позвонить)
- **Мобильные (375px, 768px)** - кликабельный tel:

#### ШАГ 5.2: Кликнуть на "Позвонить" -> открыть диалер/телефон

1. На desktop: кликнуть на "Позвонить" с номером +7 937 779-99-06

## 📌 ПУНКТ 6: Авторизация по SMS (OTP)

### Описание
Система входа в аккаунт через номер телефона с отправкой и проверкой SMS кода (OTP - One Time Password).

### Компоненты
- **PhoneAuthSheet** - модальное окно с формой входа
- **Этап 1** - ввод номера телефона
- **Этап 2** - ввод кода из SMS (6 цифр)
- **Функции** - requestBuyerSmsCode, verifyBuyerSmsCode
- **Хранение** - sessionStorage для временного хранения кода (5 минут)

### Где проверять
- **Кнопка "Вход в систему"** - в header на любой странице
- **Модальное окно** - http://localhost:3001 -> кликнуть "Вход в систему"

### Пошаговая инструкция скринкаста

#### ШАГ 6.1: Открыть модальное окно входа по SMS
```
1. Перейти на http://localhost:3001
2. Кликнуть на кнопку "Вход в систему" в header
3. Должно открыться модальное окно "Вход по номеру телефона"
4. Показать элементы:
   - [Флаг 🇷🇺] Выбор страны (по умолчанию +7 Россия)
   - Поле ввода номера телефона
   - Кнопка "Получить код" (сейчас неактивная)
   - Текст "Ваши данные защищены"
   - Согласие "Нажимая 'Получить код', вы соглашаетесь с условиями использования"
```

**Ожидаемый результат:**
- Модальное окно открывается плавно
- Все элементы видны и доступны
- На мобильных окно открывается как drawer (снизу)

#### ШАГ 6.2: Ввести номер телефона и получить SMS код
```
1. В поле номера телефона ввести: 9377799906 (или любой другой)
2. При вводе поле должно форматировать номер автоматически
   - Пример: +7 (937) 779-99-06
3. Кнопка "Получить код" должна стать активной
4. Кликнуть на кнопку "Получить код"
5. Должна появиться оповещение (toast): "Код отправлен на номер"
6. Форма должна перейти на этап 2 - ввод кода
```

**Ожидаемый результат:**
- Toast нотификация подтверждает, что код отправлен
- Поле номера скрывается, появляется поле для ввода SMS кода
- Таймер на повторную отправку (60 секунд)

#### ШАГ 6.3: Ввести SMS код и войти
```
1. После отправки номера, поле меняется на ввод 6-значного кода
2. Показать, где найти SMS код в консоли (для разработки):
   - Открыть DevTools -> Console
   - Там должен быть лог: "[SMS Code - Development Only]: XXXXXX"
   - Скопировать 6 цифр
3. Вернуться в модальное окно
4. Вставить код в поле "Введите код из SMS"
5. При вводе 6-го символа или кликнути кнопки "Войти":
   - Должна появиться оповещение: "Вы успешно вошли"
   - Модальное окно должно закрыться
   - В header должна отобразиться информация пользователя
```

**Ожидаемый результат:**
- Успешная авторизация
- Пользователь видит свой профиль/имя в header или личный кабинет

#### ШАГ 6.4: Показать повторную отправку кода
```
1. Открыть вход по SMS снова
2. Ввести номер и кликнуть "Получить код"
3. После отправки кода, под полем ввода код должна быть кнопка:
   "Отправить код снова (ждите 60 сек)"
4. Кнопка должна быть неактивна с обратным отсчетом
5. Ждать 60 секунд или пока отсчет не закончится
6. Кнопка должна стать активной и позволить отправить код заново
```

**Ожидаемый результат:**

## 📌 ПУНКТ 7: Лимиты/Антифрод, TTL кода, Защита от брутфорса

### Описание
Безопасность процесса SMS авторизации:
- **TTL кода** - код действует 5 минут (300 секунд)
- **Лимит попыток** - максимум 3 неверных кода за сессию
- **Таймер повторной отправки** - 60 секунд между отправками
- **Защита от брутфорса** - код удаляется из памяти при истечении TTL

### Где проверять
- **sessionStorage** - время истечения кода
- **Таймер** - 60 секунд между отправками
- **Попытки ввода** - максимум 3 неверных кода

### Пошаговая инструкция скринкаста

#### ШАГ 7.1: Проверить TTL кода (5 минут)
```
1. Открыть вход по SMS и получить код
2. Открыть DevTools -> Console
3. Выполнить: JSON.parse(sessionStorage.getItem('sms_verification'))
4. Увидеть объект с полями:
   {
     "code": "XXXXXX",
     "phone": "79377799906",
     "expiresAt": TIMESTAMP (текущее время + 5 минут)
   }
5. Проверить разницу: (expiresAt - Date.now()) / 1000 / 60
   - Результат должен быть ~5 минут
```

**Ожидаемый результат:**
- Время истечения = текущее время + 300 000 ms (5 минут)

#### ШАГ 7.2: Проверить таймер повторной отправки (60 секунд)
```
1. Ввести номер, кликнуть "Получить код"
2. После отправки, под полем должна появиться кнопка:
   "Отправить код снова (ждите 59 сек)"
3. Время должно отсчитываться вниз: 59, 58, 57...
4. После достижения 0 кнопка становится активной
5. Открыть DevTools -> Console и проверить:
   const auth = useAuthStore.getState();
   console.log('Время отправки:', new Date(auth.smsCodeSentAt));
   console.log('Прошло секунд:', Math.floor((Date.now() - auth.smsCodeSentAt) / 1000));
```

**Ожидаемый результат:**
- Таймер 60 секунд работает корректно
- После истечения таймера кнопка "Отправить заново" становится активной

#### ШАГ 7.3: Проверить удаление кода при истечении TTL
```
1. Получить код
2. Записать себе время: current time = Date.now()
3. Ждать 5 минут (или эмулировать переход времени в консоли)
4. Попытаться ввести код (даже правильный)
5. Должна появиться ошибка: "SMS код истёк. Запросите новый код"
6. Код должен удалиться из sessionStorage
```

**Ожидаемый результат:**
- По истечении 5 минут код перестает работать
- sessionStorage.getItem('sms_verification') возвращает null

#### ШАГ 7.4: Показать защиту от брутфорса (3 попытки)
```
1. Получить SMS код
2. Попытка 1: ввести неправильный код (например, 000000) -> ошибка
3. Попытка 2: ввести неправильный код (например, 111111) -> ошибка
4. Попытка 3: ввести неправильный код (например, 222222) -> ошибка
5. После 3-й попытки должна блокировка или требование повторной отправки кода
```

**Ожидаемый результат:**
- Система не позволяет бесконечно перебирать коды
- После определенного количества попыток требуется новый код

## 📌 ПУНКТ 8: Метрики в товар (Продажи, Просмотры, Цена, Рейтинг)

### Описание
На карточке товара в выдаче отображаются ключевые метрики:
1. **Всего заказали** - количество продаж (продано X)
2. **Сейчас в карточке** - текущее количество просмотров (НА САЙТЕ)
3. **Всего просмотров** - общее количество просмотров (за все время)
4. **Цена выросла/упала** - стрелка вверх/вниз и процент изменения
5. **Рейтинг** - звездочка и оценка (от 1 до 5)

### Где проверять
- **Страница товаров** - http://localhost:3001/products (если товары загружаются)
- **Карточка товара на главной** - http://localhost:3001 (в разделах с рекомендациями)
- **Страница товара** - http://localhost:3001/product/[id]

### Пошаговая инструкция скринкаста

#### ШАГ 8.1: Показать метрики на карточке товара
```
1. Перейти на http://localhost:3001/products
2. Если товары не загружаются - перейти на http://localhost:3001 и скроллить к рекомендациям
3. Найти карточку товара с информацией
4. На карточке должны быть видны:
   
   [ИЗОБРАЖЕНИЕ ТОВАРА]
   
   Название товара (например, "Букет из пионов")
   5 прод.  |  1000 просмотров  |  в наличии 50     <- МЕТРИКИ
   Категория (например, "Цветы")
   4 999₽  ⬆️ 15%                                      <- ЦЕНА И ТРЕНД
   ⭐ 4.5                                              <- РЕЙТИНГ
   [В корзину]
```

**Ожидаемый результат:**
- Все метрики четко видны на карточке
- Информация понятно организована
- Цвета контрастные (все видно на фото товара)

#### ШАГ 8.2: Разобрать каждую метрику
```
МЕТРИКА 1 - "Всего заказали" (Продажи)
  Текст: "5 прод." (5 произведено/продано)
  Источник: поле total_sold из API
  Показывается: если значение > 0
  
МЕТРИКА 2 - "Всего просмотров"
  Текст: "1000 просмотров"
  Источник: поле view_count из API
  Показывается: если значение > 0
  
МЕТРИКА 3 - "Сейчас в карточке" (На складе/В наличии)
  Текст: "в наличии 50"
  Источник: поле current_amount из API
  Показывается: если значение > 0
  Информирует: сколько единиц товара сейчас доступно
  
МЕТРИКА 4 - "Цена выросла/упала"
  Вверх ⬆️ красный цвет: цена выросла (например, +15%)
  Вниз ⬇️ зеленый цвет: цена упала (например, -10%)
  Источник: price_change_percent или вычисляется из previous_price
  Показывается: только если есть изменение цены
  
МЕТРИКА 5 - "Рейтинг"
  Символ: ⭐ звезда
  Значение: 4.5 (от 1 до 5)
  Источник: avg_rating из API
  Показывается: только если значение > 0
```

**Ожидаемый результат:**
- Пользователь сразу видит популярность товара (продажи, просмотры)
- Видит наличие товара
- Видит тренд цены
- Видит оценку других покупателей
