# Инструкции для скринкастов — 5 пунктов ТЗ

## Пункт 1: SEO ("Все для SEO")

### ШАГИ 1.1: Title и Meta Description

**Шаг 1.1.1 — Откройте главную:**
- Перейдите по ссылке: http://localhost:3001/
- Страница должна загрузиться с заголовком в браузерной вкладке

**Шаг 1.1.2 — Откройте DevTools:**
- Нажмите **F12** (или Ctrl+Shift+I)
- Перейдите на вкладку **Elements** (вверху слева)

**Шаг 1.1.3 — Найдите Title:**
- В левой части DevTools будет дерево HTML
- Прокрутите вверх до тега `<head>`
- Внутри `<head>` найдите `<title>БыстроИточка - маркетплейс с быстрой доставкой | быстроИточка</title>`
- **Скриншот:** выделите весь тег `<title>`

**Шаг 1.1.4 — Найдите Meta Description:**
- В этом же `<head>` ищите строку с `<meta name="description"`
- Полный текст: `<meta name="description" content="БыстроИточка - маркетплейс с быстрой доставкой. Категории товаров, рекомендации и выгодные предложения в одном месте.">`
- **Скриншот:** выделите весь тег `<meta name="description">`

**Шаг 1.1.5 — Проверьте в коде:**
- Используйте Ctrl+F в DevTools
- Введите: `"БыстроИточка - маркетплейс с быстрой доставкой"`
- DevTools подсветит все совпадения (должны быть в title и в meta description)

### ШАГИ 1.2: H1 теги на всех страницах

**Шаг 1.2.1 — H1 на главной (скрытый):**
- Останьтесь на http://localhost:3001/
- DevTools → Elements (уже открыт)
- Используйте Ctrl+F для поиска в DevTools
- Введите: `<h1`
- DevTools найдёт h1 теги
- Вы увидите: `<h1 class="sr-only">БыстроИточка - маркетплейс с быстрой доставкой</h1>`
- Этот h1 скрытый (sr-only = screen-reader only), но видно в Elements
- **Скриншот:** найденный h1 в DevTools

**Шаг 1.2.2 — H1 на странице товаров:**
- Перейдите по ссылке: http://localhost:3001/products
- DevTools → Elements → Ctrl+F
- Ищите: `<h1`
- Вы найдёте: `<h1 class="text-lg font-medium tracking-tight">Категория(...)</h1>`
- Это основной заголовок со счётчиком товаров
- **Скриншот:** h1 с категорией и числом товаров

**Шаг 1.2.3 — H1 на странице одного товара:**
- Перейдите по ссылке: http://localhost:3001/product/111656
- DevTools → Elements → Ctrl+F
- Ищите: `<h1`
- Вы найдёте: `<h1 class="text-xl font-medium tracking-tight">Тюльпан</h1>`
- Это название товара как h1
- **Скриншот:** h1 с названием товара

### ШАГИ 1.3: Canonical ссылки

**Шаг 1.3.1 — Canonical на главной:**
- Станьте на http://localhost:3001/ (главная)
- DevTools → Elements → Ctrl+F
- Ищите: `rel="canonical"`
- Вы найдёте: `<link rel="canonical" href="/"/>`
- Это говорит поисковикам, что это главная страница
- **Скриншот:** canonical тег

**Шаг 1.3.2 — Canonical на /products:**
- Перейдите на: http://localhost:3001/products
- DevTools → Elements → Ctrl+F
- Ищите: `rel="canonical"`
- Вы найдёте: `<link rel="canonical" href="/products"/>`
- **Скриншот:** canonical тег

**Шаг 1.3.3 — Canonical на товаре:**
- Перейдите на: http://localhost:3001/product/111656
- DevTools → Elements → Ctrl+F
- Ищите: `rel="canonical"`
- Вы найдёте: `<link rel="canonical" href="/product/111656"/>`
- **Скриншот:** canonical тег с ID товара

### ШАГИ 1.4: Robots метатег

**Шаг 1.4.1 — Robots на главной (индексируется):**
- Останьтесь на http://localhost:3001/
- DevTools → Elements → Ctrl+F
- Ищите: `name="robots"`
- Вы найдёте: `<meta name="robots" content="index, follow">`
- Это говорит поисковикам: индексируй эту страницу и переходи по ссылкам
- **Скриншот:** robots метатег

**Шаг 1.4.2 — Robots на /products с фильтрами (не индексируется):**
- Перейдите на: http://localhost:3001/products?sort=price&filter=rating
- (добавьте любые фильтры в URL)
- DevTools → Elements → Ctrl+F
- Ищите: `name="robots"`
- Вы найдёте: `<meta name="robots" content="noindex, follow">`
- Это говорит поисковикам: НЕ индексируй эту страницу (т.к. это отфильтрованная версия)
- **Скриншот:** robots метатег с noindex

**Шаг 1.4.3 — Robots на товаре (индексируется):**
- Перейдите на: http://localhost:3001/product/111656
- DevTools → Elements → Ctrl+F
- Ищите: `name="robots"`
- Вы найдёте: `<meta name="robots" content="index, follow">`
- **Скриншот:** robots метатег

### ШАГИ 1.5: Sitemap.xml

**Шаг 1.5.1 — Откройте sitemap:**
- В адресной строке введите: http://localhost:3001/sitemap.xml
- Страница откроется с XML содержимым
- Вы увидите примерно такое:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset>
  <url>
    <loc>https://bystroi.ru/</loc>
    <lastModified>2026-04-30</lastModified>
    <changeFrequency>daily</changeFrequency>
    <priority>1</priority>
  </url>
  <url>
    <loc>https://bystroi.ru/products</loc>
    ...
  </url>
</urlset>
```

**Шаг 1.5.2 — Проверьте основные URL'ы:**
- В sitemap должны быть:
  - ✓ `https://bystroi.ru/` (главная, priority 1)
  - ✓ `https://bystroi.ru/products` (товары, priority 0.9)
  - ✓ `https://bystroi.ru/rating` (рейтинг, priority 0.8)
  - ✓ `https://bystroi.ru/categories` (категории)
  - ✓ `https://bystroi.ru/favorites` (избранное)
  - ✓ `https://bystroi.ru/terms` (условия)
  - ✓ `https://bystroi.ru/privacy` (конфиденциальность)

**Шаг 1.5.3 — Скриншот:**
- Выделите весь content XML (Ctrl+A)
- Сделайте скриншот с видимыми URL'ами

## Пункт 2: Баннеры + контейнер по масштабу

### ШАГИ 2.1: Адаптивные размеры баннеров

**Шаг 2.1.1 — Откройте главную:**
- Перейдите на: http://localhost:3001/
- Страница должна полностью загрузиться

**Шаг 2.1.2 — Откройте Responsive Design Mode:**
- Нажмите **Ctrl+Shift+M** (или F12 → иконка телефона в левом углу DevTools)
- Вверху появится выпадающий список размеров экранов

**Шаг 2.1.3 — Проверьте на мобильном (375px):**
- В dropdown выберите или введите: **375 × 667** (iPhone SE)
- Вы увидите баннеры в узком формате
- Баннеры должны быть:
  - Высота примерно **220px**
  - Ширина **100% экрана** (без горизонтального скролла)
  - Текст читаемый: "Выгодные предложения каждый день", "Быстрый заказ без лишних шагов"
  - Изображение масштабируется без искажений
  - Внизу видны **точки для переключения** (pagination dots)
- **Скриншот:** полный экран баннера на 375px

**Шаг 2.1.4 — Проверьте на планшете (768px):**
- В dropdown выберите: **768 × 1024** (iPad)
- Вы увидите:
  - Высота баннера примерно **300px**
  - Больше текста в одной строке
  - Видно **1.3 баннера за раз** (левый край следующего баннера виден)
- **Скриншот:** баннеры на 768px

**Шаг 2.1.5 — Проверьте на десктопе (1440px):**
- В dropdown выберите: **1440 × 900** (стандартный ноутбук)
- Вы увидите:
  - Высота баннера примерно **340px**
  - **2+ баннера видны одновременно**
  - Полный контент всех слайдов
- **Скриншот:** баннеры на 1440px

**Шаг 2.1.6 — Проверьте отсутствие горизонтального скролла:**
- На каждом размере экрана
- Пролистайте вверх-вниз (вертикальный скролл OK)
- **НЕ должно быть горизонтального скролла** (влево-вправо)
- Баннеры всегда занимают ровно 100% ширины экрана
- **Скриншот:** DevTools показывает ширину 375px (без overflow)

### ШАГИ 2.2: Lazy-load изображений баннеров

**Шаг 2.2.1 — Откройте DevTools на главной:**
- Останьтесь на http://localhost:3001/
- Нажмите **F12**
- Откройте вкладку **Network** (вверху вкладок DevTools)

**Шаг 2.2.2 — Перезагрузите страницу с DevTools открытым:**
- Нажмите **Ctrl+Shift+R** (жёсткая перезагрузка)
- Network будет показывать все загрузки

**Шаг 2.2.3 — Найдите изображения баннеров:**
- В Network ищите запросы с `/_next/image/` или `poster/`
- Должны быть 4 изображения баннеров (poster/1.png, poster/2.png, poster/3.png, poster/4.png)
- Первые 2 должны загружаться **сразу** после загрузки страницы
- Следующие 2 могут быть **не загружены** (они видны дальше вниз по странице)

**Шаг 2.2.4 — Пролистайте страницу вниз:**
- Пока DevTools Network открыта
- Медленно пролистайте вниз до баннеров 3 и 4
- В Network вы увидите, как загружаются новые изображения **по мере того, как они становятся видны**
- Это lazy-load: изображение загружается только когда пользователь его видит

**Шаг 2.2.5 — Скриншот:**
- Network tab с видимыми запросами изображений баннеров
- Можно отфильтровать по "Img" для наглядности (слева внизу опция "Img")

### ШАГИ 2.3: Pagination (переключение баннеров)

**Шаг 2.3.1 — Откройте главную в обычном режиме:**
- http://localhost:3001/
- Закройте Responsive Mode (Ctrl+Shift+M) — вернёмся к нормальному размеру
- Закройте DevTools (F12)

**Шаг 2.3.2 — Найдите баннеры:**
- В верхней части главной вы увидите 4 больших баннера
- Баннер 1: "Выгодные предложения каждый день"
- Баннер 2: "Быстрый заказ без лишних шагов"
- Баннер 3: "Товары и подарки для любого повода"
- Баннер 4: "Собирайте покупки быстрее..."

**Шаг 2.3.3 — Найдите точки (dots) внизу баннеров:**
- Внизу баннеров есть **маленькие точки** (обычно белые или серые)
- Каждая точка соответствует одному баннеру
- Точки обычно расположены горизонтально в центре внизу

**Шаг 2.3.4 — Кликните на вторую точку:**
- Нажмите на вторую точку (вторая справа)
- Баннер **плавно переместится** к баннеру 2
- Вторая точка станет **активной/выделенной** (другого цвета)

**Шаг 2.3.5 — Кликните на третью точку:**
- Нажмите на третью точку
- Баннер переместится к баннеру 3
- **Скриншот:** баннер 3 с активной третьей точкой

**Шаг 2.3.6 — Проверьте на мобильном:**
- Снова откройте Responsive Mode (Ctrl+Shift+M)
- Выберите 375px
- Видна только 1 точка-баннер за раз
- На планшете (768px) видно 1.3 баннера (левый край следующего виден)
- **Скриншот:** мобильный вид с точками

## Пункт 3: Заказ в 1 клик

### ШАГИ 3.1: Кнопка "Купить в 1 клик" на странице товара

**Шаг 3.1.1 — Откройте страницу товара:**
- Перейдите по ссылке: http://localhost:3001/product/111656
- Страница должна загрузиться (товар "Тюльпан")

**Шаг 3.1.2 — Найдите две кнопки добавления:**
- Пролистайте страницу вниз немного (примерно на половину экрана)
- В правой части (или в конце контента) найдите блок "Оформить заказ"
- Там должны быть **2 синие кнопки рядом:**
  - **Левая кнопка:** "В корзину" (или иконка корзины + текст)
  - **Правая кнопка:** "Купить в 1 клик"

**Шаг 3.1.3 — Скриншот:**
- Выделите и сфотографируйте обе кнопки
- Они должны быть расположены рядом горизонтально

### ШАГИ 3.2: Открытие модального окна быстрого заказа

**Шаг 3.2.1 — Нажмите на "Купить в 1 клик":**
- На странице товара нажмите кнопку **"Купить в 1 клик"**
- Должно появиться **модальное окно** (popup/dialog) в центре экрана

**Шаг 3.2.2 — Проверьте заголовок модального окна:**
- Вверху модального окна должен быть заголовок: **"Купить в 1 клик"** или похожий
- Справа вверху модального окна есть кнопка **X** для закрытия

**Шаг 3.2.3 — Скриншот:**
- Сфотографируйте всё модальное окно целиком

### ШАГИ 3.3: Заполнение формы быстрого заказа

**Шаг 3.3.1 — Посмотрите поля формы:**
- В модальном окне должны быть следующие поля:

1. **Поле "Имя"** (Name field):
   - Обычно первое поле
   - Может быть предзаполнено (если вы вошли в систему)
   - Если нет — пусто

2. **Поле "Телефон"** (Phone field):
   - Важное обязательное поле
   - Должно быть пусто или с предзаполненным номером
   - Формат: +7 ХХХ ХХХ-ХХ-ХХ или просто цифры

3. **Поле "Адрес"** (Address field):
   - Обязательное поле
   - Может быть предзаполнено из сохранённых данных
   - Если нет — нужно ввести вручную

4. **Поле "Количество"** (Quantity field):
   - Обычно со значением по умолчанию "1"
   - Рядом два кнопки: **"-"** (уменьшить) и **"+"** (увеличить)
   - Можно также вводить число вручную

5. **Чекбокс согласия** (Agreement checkbox):
   - "Я согласен с условиями доставки" или похожий текст
   - Должен быть отмечен (checked)

6. **Кнопка "Заказать"** (Order button):
   - Большая синяя кнопка внизу формы
   - **Заказать** или **Submit Order**

**Шаг 3.3.2 — Скриншот:**
- Сфотографируйте форму со всеми полями видимыми

### ШАГИ 3.4: Заполнение и отправка заказа

**Шаг 3.4.1 — Заполните поле "Имя":**
- Если пусто — введите любое имя: `Тест` или `Иван`
- Это обязательное поле

**Шаг 3.4.2 — Заполните поле "Телефон":**
- Нажмите на поле "Телефон"
- Введите номер: `79377799906` или `+79377799906`
- Это обязательное поле

**Шаг 3.4.3 — Заполните поле "Адрес":**
- Нажмите на поле "Адрес"
- Введите адрес: `Москва, ул. Примерная, д. 1`
- Это обязательное поле

**Шаг 3.4.4 — Установите количество:**
- Проверьте, что в поле количество стоит хотя бы "1"
- Можете увеличить нажав кнопку "+" (если нужно)

**Шаг 3.4.5 — Отметьте согласие:**
- Убедитесь, что чекбокс согласия **отмечен** ✓

**Шаг 3.4.6 — Нажмите "Заказать":**
- Кликните на кнопку **"Заказать"** внизу формы
- Форма должна отправиться на backend

**Шаг 3.4.7 — Скриншот:**
- Скриншот заполненной формы **перед** нажатием кнопки
- Скриншот **после** отправки (сообщение об успехе или закрытие)

## Пункт 4: Ускорение выдачи + поиск по радиусу

### ШАГИ 4.1: Параметры API для радиуса

**Шаг 4.1.1 — Откройте страницу товаров:**
- Перейдите по ссылке: http://localhost:3001/products
- Страница товаров должна загрузиться

**Шаг 4.1.2 — Откройте DevTools:**
- Нажмите **F12**
- Перейдите на вкладку **Network** (вверху вкладок)

**Шаг 4.1.3 — Перезагрузите страницу:**
- Нажмите **Ctrl+Shift+R** (жёсткая перезагрузка)
- Network начнёт показывать все запросы

**Шаг 4.1.4 — Найдите API запрос:**
- В Network ищите запросы к `app.tablecrm.com`
- Должен быть запрос: `https://app.tablecrm.com/api/v1/mp/products?...`
- Нажмите на этот запрос, чтобы его выделить

**Шаг 4.1.5 — Посмотрите Query Parameters:**
- В нижней части DevTools (когда запрос выбран) перейдите на вкладку **Headers** или **Request**
- Ищите раздел **Query String Parameters** или просто посмотрите URL
- Вы должны увидеть параметры:
  - `size=100` (максимальное количество товаров)
  - `page=1` (первая страница)
  - `sort_by=total_sold` или другой вариант
  - `sort_order=desc`

**Шаг 4.1.6 — Скриншот:**
- Выделите весь URL запроса или Query Parameters
- На скриншоте должно быть видно, что `size=100`

### ШАГИ 4.2: Селектор радиуса

**Шаг 4.2.1 — Посмотрите на верхнюю часть страницы:**
- На странице /products вверху справа ищите dropdown с текстом **"Радиус:"**
- Должно быть написано что-то вроде:
  - "Радиус: 20 км" (по умолчанию)
  - или просто иконка с выпадающим меню

**Шаг 4.2.2 — Кликните на dropdown радиуса:**
- Нажмите на dropdown "Радиус: 20 км"
- Откроется список вариантов:
  - 3 км
  - 5 км
  - 10 км
  - 20 км (текущий, выделен)
  - 50 км

**Шаг 4.2.3 — Выберите "Радиус: 10 км":**
- Нажмите на вариант **"10 км"**
- URL должен измениться на: `...?radius_km=10&apply_radius_filter=true...`
- Товары на странице должны перезагрузиться

**Шаг 4.2.4 — Посмотрите на Network:**
- В DevTools Network должен появиться новый запрос к API
- Он должен содержать новые параметры:
  - `radius_km=10` (вместо 20)
  - `apply_radius_filter=true`
  - `lat=...` и `lon=...` (координаты вашего города)

**Шаг 4.2.5 — Скриншот:**
- Dropdown с видимыми вариантами радиуса
- Network tab с новым запросом и параметрами

### ШАГИ 4.3: Проверка производительности

**Шаг 4.3.1 — Посмотрите на время загрузки:**
- В Network tab выделите запрос к API
- В нижней части DevTools посмотрите на столбец **Time** или **Duration**
