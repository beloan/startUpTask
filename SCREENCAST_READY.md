## ✅ Пункт 1: SEO ("Все для SEO")

**Статус:** ✓ ГОТОВО

### Что проверить в браузере:
1. **Главная страница:**
   - DevTools → Elements → заголовок содержит `<title>` и `<meta name="description">`
   - URL: http://localhost:3001/

2. **H1 теги:**
   - На главной: скрытый h1 (sr-only) с названием маркетплейса
   - На /products: `"Категория(...)"` с количеством товаров
   - На /product/111656: названии товара `"Тюльпан"`

3. **Canonical ссылки:**
   - Главная: `<link rel="canonical" href="/"/>`
   - /products: `<link rel="canonical" href="/products"/>`
   - /product/[id]: `<link rel="canonical" href="/product/111656"/>`

4. **robots.txt (динамический):**
   - Доступен через `/robots.ts` Next.js API
   - Disallow: /admin/, /api/, /cart/, /search/, /account/, /favorites/, /statistics/, /payment/

5. **sitemap.xml:**
   - http://localhost:3001/sitemap.xml (работает)
   - Содержит: /, /products/, /rating/, /categories/, /favorites/, /terms/, /privacy/

### Команды для проверки:
```powershell
# Проверить сайтсеп
Invoke-WebRequest http://localhost:3001/sitemap.xml

# Проверить robots.ts (генерируется)
# Он работает в Next.js автоматически
```

## ✅ Пункт 3: Заказ в 1 клик

**Статус:** ✓ ГОТОВО

1. **Откройте карточку товара:**

5. **Отправьте заказ:**
   - Кликните "Заказать"
   - После успеха: уведомление в tost-е

## ✅ Пункт 4: Ускорение выдачи + поиск по радиусу

**Статус:** ✓ ГОТОВО

### Что проверить в браузере:
1. **Откройте /products:**
   - http://localhost:3001/products

2. **Найдите селектор радиуса:**
   - Вверху справа есть dropdown "Радиус: 20 км"
   - Варианты: 3 км, 5 км, 10 км, 20 км, 50 км

3. **Измените радиус:**
   - Выберите "Радиус: 10 км"
   - URL обновится: `?radius_km=10&apply_radius_filter=true`

4. **Проверьте API параметры:**
   - DevTools → Network → XHR
   - Ищите запросы к `app.tablecrm.com/api/v1/mp/products`
   - Параметры должны содержать:
     - `apply_radius_filter=true`
     - `radius_km=10` (или выбранное значение)
     - `lat=...` и `lon=...` (координаты)
     - `size=100` (максимум по API ограничению)

5. **Проверьте кэширование:**
   - При повторном клике на тот же радиус → запрос из кэша
   - Network → проверьте Status код (должен быть 200 или из памяти)

### Ключевые файлы:
- [src/app/(routes)/products/ProductsPageContent.tsx](src/app/(routes)/products/ProductsPageContent.tsx) — страница товаров с фильтром радиуса
- API лимит: `size ≤ 100`

## ✅ Пункт 5: Рекомендации — сортировка (новые vs рейтинг)

**Статус:** ✓ ГОТОВО

### Что проверить в браузере:
1. **Откройте главную:**
   - http://localhost:3001/

2. **Пролистайте вниз до "Рекомендации по основным вашим просмотрам":**
   - Найдите две кнопки:
     - **"Сначала последние"** (активная по умолчанию)
     - **"По рейтингу"**

3. **Нажмите "Сначала последние":**
   - Товары переупорядочиваются по дате создания (новые сверху)
   - DevTools → Network → XHR
   - API параметры: `sort_by=created_at&sort_order=desc`

4. **Нажмите "По рейтингу":**
   - Товары переупорядочиваются по рейтингу (высокие сверху)
   - DevTools → Network → XHR
   - API параметры: `sort_by=rating&sort_order=desc`

5. **Проверьте вторую секцию рекомендаций:**
   - Пролистайте дальше вниз до "Лучшие предложения для вас"
   - Там также две кнопки сортировки
   - Они работают независимо от первой секции

### Ключевые файлы:
- [src/widgets/recommendations/index.tsx](src/widgets/recommendations/index.tsx) — компонент рекомендаций с двумя сортировками
